# Parametric Curve Parameter Estimation (θ, M, X)

## What this solves
The assignment gives a parametric curve of the form:

- **x(t)** = t·cos(θ) − e^(M|t|)·sin(0.3t)·sin(θ) + X
- **y(t)** = 42 + t·sin(θ) + e^(M|t|)·sin(0.3t)·cos(θ)

The unknown parameters are **θ, M, X**. The dataset **xy_data.csv** contains points lying on the curve. The goal is to estimate the parameters so that the curve generated by the model matches the CSV points.

## Approach (engineering-style)
This is formulated as a numerical parameter estimation problem. Since only three parameters (θ, M and X) are unknown, optimization is more appropriate than machine learning.

I treated it as **parameter estimation**:


1. **Load and inspect the data** (read `xy_data.csv`, check columns `x` and `y`).
2. **Map CSV points to parameter values t**: the assignment states the curve uses values of **t in [6, 60]**. With no explicit t column in the CSV, I assumed the CSV points correspond to **uniformly sampled t** over that interval.
3. **Define an error metric**: I used the assignment’s idea of **L1 distance** between predicted and observed coordinates:
   
   `L1 = Σ (|x_pred − x_obs| + |y_pred − y_obs|)`
4. **Coarse search**:
- Loop over a grid for **θ** within **(0°, 50°)**

- Loop over a grid for **M** within **(-0.05, 0.05)**

- For each (θ, M), **choose X analytically** using the fact that x(t) shifts by +X (with **0 < X < 100** as stated in the assignment). This makes the coarse search fast and explainable.

5. **Numerical refinement**:
   - Starting from the best grid candidate, I refine with a numerical optimizer using a smooth Huber-style surrogate to handle the non-smooth nature of L1.
6. **Validation**:
   - Plot CSV points vs fitted curve and save the figures.

## How to run
1. Put `xy_data.csv` in the project folder.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Run:
   ```bash
   python solution.py
   ```

The script prints the estimated parameters (θ, M, X) and the final L1 error.

## Notes
- The optimizer portion may take a few seconds depending on your machine and the number of CSV points.
- The t-uniform sampling assumption is necessary unless the CSV explicitly provides t values.
